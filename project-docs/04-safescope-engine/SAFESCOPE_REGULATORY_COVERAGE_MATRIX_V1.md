# SafeScope Regulatory Coverage Matrix v1

This document outlines the architecture and purpose of the `RegulatoryCoverageMatrixService`, which produces a deterministic audit of SafeScope's current regulatory coverage against a predefined list of required core safety standards.

## Purpose
To move SafeScope toward a mature, audit-ready safety AI classification, we must be able to empirically demonstrate what core regulatory standards are actively covered by the engine, which are staged for review (draft), and which are missing entirely.

The coverage matrix serves as the source of truth for tracking the engine's regulatory knowledge completeness.

## Operation
The `RegulatoryCoverageMatrixService` runs as a deterministic script. It performs the following steps:
1.  **Reads the Local Inventory:** It consumes the `RegulatorySourceInventory` generated by the `RegulatorySourceAuditService`.
2.  **Analyzes Core Requirements:** It iterates through a predefined list of `CORE_STANDARD_REQUIREMENTS` (e.g., OSHA 1910.212, MSHA 56.14).
3.  **Classifies Coverage:** For each requirement, it checks the local inventory's normalized citations to determine if the standard is `approved`, `draft`, or `missing`.
4.  **Surfaces Unknowns:** It identifies records with missing or unknown metadata (e.g., unknown agency, unknown jurisdiction) to flag them for cleanup.
5.  **Generates Matrix:** It outputs a JSON report to `safescope-data/source-audit/regulatory-coverage-matrix-v1.json`.

## Governance Boundaries
-   **Read-Only:** The matrix generator is strictly read-only. It audits the existing registry and draft candidates.
-   **No Auto-Promotion:** Identifying a gap or a draft candidate does NOT automatically promote the candidate to approved knowledge. All promotion remains subject to the established Role-Based Approval Gates.
-   **Deterministic Validation:** The matrix relies on local fixture data and the current repository state, ensuring deterministic validation without live network calls.

## Output Structure
The generated `regulatory-coverage-matrix-v1.json` includes:
-   **Summary Totals:** High-level counts of approved, draft, and missing core standards.
-   **Coverage Distributions:** Breakdowns by agency, jurisdiction, authority tier, and hazard family.
-   **Core Standards Coverage:** Detailed status for each required core standard.
-   **Unknown Metadata Records:** A list of records requiring metadata remediation.
-   **Duplicate/Overlap Candidates:** A mapping of citations to multiple records, indicating potential overlaps requiring review.
