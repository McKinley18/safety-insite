# SafeScope Regulatory Source Audit + Differential Ingestion Engine v1

This document outlines the architecture and governance rules for the automated ingestion and auditing of official regulatory sources.

## Purpose
The SafeScope Regulatory Source Audit and Ingestion Engine ensures that SafeScope's internal knowledge base remains aligned with official public safety standards (e.g., OSHA, MSHA) without introducing unverified or hallucinated content. It functions as a secure bridge between external data sources and the governed `ReviewerCandidateConsole`.

## Core Components

1. **Regulatory Source Audit Service**
   - Scans the existing `safescope-data/approved-knowledge/registry/` and `draft-candidates/`.
   - Generates an `InventoryReport` detailing coverage by agency, jurisdiction, hazard family, and citation.
   - Identifies metadata gaps and overlaps.

2. **Official Source Connectors**
   - Interfaces (`IRegulatorySourceConnector`) designed to fetch data from authoritative sources.
   - Supported source systems:
     - `ecfr` (e.g., OSHA 1910, 1926, MSHA 56, 57)
     - `osha_fatality`, `msha_fatality` (Alerts and lesson summaries)
     - `osha_investigation` (Enforcement context)
   - *Note: V1 utilizes local-first fixture snapshots to verify logic without making live network calls.*

3. **Differential Comparison Service**
   - Compares incoming connector results against the local inventory using normalized citations (e.g., `1910.212(a)(1)` -> `1910.212`).
   - Classifies candidates to prevent duplicates:
     - `already_covered`: Exists and is up-to-date.
     - `exact_duplicate`: Ignored.
     - `source_changed_review_required`: Existing citation with a newer revision date.
     - `missing_from_safescope`: New regulatory requirement.
     - `supplemental_candidate`: Fatality or investigation context.
     - `rejected_unknown_authority`: Low quality or unverified source.

4. **Governed Ingestion Adapter**
   - Converts approved differentials into `ReviewerCandidate` records.
   - Stages candidates in the `ReviewerCandidateConsole` (status: `pending_review`).
   - **Crucial:** It never automatically promotes scraped content to `approved knowledge`.

## Governance Boundaries
- **Advisory Only:** All ingested candidates inherit `advisoryOnly: true`.
- **No Violation Declarations:** The engine strictly prohibits the generation or ingestion of text containing "is a violation" or definitive legal declarations.
- **Qualified Human Review:** Every candidate generated by this engine requires approval by an authorized role (e.g., `compliance_admin`, `osha_general_industry_reviewer`) before affecting classification logic.

## Future Live Connector Plan
Future iterations will replace local fixtures with live API connections to the eCFR and agency data portals. The differential comparison and human review gates ensure this transition can happen safely without corrupting the production database.
